# soia-pkm-alipan-curator — 云盘资源顾问

## 定位与依赖

对标 huashu-weread-advisor 之于 weread-skills：底层命令全部走 [[soia-pkm-alipan]]，本 skill 只管**方法论和产出**。
学习计划的排期方法论继承 soia-pkm-reading-plan：**书按字数排期 → 视频按 集数×单集时长 排期**，不拍脑袋。

## 数据源（三件套，plan 前必读）

1. **资源地图**：OB 里的《阿里云盘索引》（`20_资料库/阿里云盘索引.md`）——只推**云盘里已有**的资源，别让用户去找新资源（对标 reading-plan 的"别让用户重复买"）
2. **孩子画像**：memory `kids-learning-profiles` + 用户本次给的成绩单/截图——多孩家庭必须**对号入座**（哪个孩子、什么学段、弱在哪）
3. **计划格式**：参照 `references/plan-template.md`（仿用户阅读计划格式：frontmatter + 映射总表 + 排期 + 待确认转正）

## 子命令路由

### inventory — 盘点
逐层 `ls` 扫描（顶层→四级），产出核对清单到 OB `10_工作台/云盘深度核对清单.md`：每支记「内容 · 类型 · 建议动作」，用户拍板后交 organize。

### organize — 整理
执行规范（每步终态验证，安全守则见 soia-pkm-alipan）：
- **去营销尾巴**：目录名里的「公众号：XXX」「【xx.xGB】」「持续更新」等广告字样批量 rename 掉
- **去套娃**：目录内只有一个子目录 → 内容上提一层、删壳
- **删广告**：「↑↑订阅↑↑」目录、`800T资源.txt`、`福利码.txt` 之类直接 rm
- **按人群分层**：多孩家庭学习资源按学段分（幼儿启蒙/小学/共享），不按单个孩子命名
- **命名统一**：同级目录编号格式一致（如 `01-06` 前缀），利于排序

### catalog — 索引落 OB
把定稿结构写成《阿里云盘索引》：双盘分区、每支一行带内容摘要、待办区、精确跳转说明（alipan.com 的 folder file_id 拼链接）。整理后**必须**同步索引，否则索引腐烂。

### plan — 学习计划（高频主场景）
输入：成绩截图/学情描述（如「这是老大的期末，出个暑假方案」）。流程：
1. **诊断**：从成绩单提取 分数/排名/失分点/目标（截图里有就直接用，没有就问 1-2 个关键问题）
2. **资源映射**：弱项 → 云盘索引里的对应资源，标注「为什么这个资源治这个失分点」
3. **排期**：视频资源查实**集数与单集时长**（云盘 ls 数一下），按「每周 X 次 × 每次 Y 分钟」倒排出周计划表；一个暑假别超过 3 条主线（学科2 + 兴趣/体育1）
4. **落盘**：写到 OB `40_阅读与摘抄/40_孩子书库/学习计划/YYYY-MM-DD-<孩子>-<主题>.md`，状态「待确认」，用户拍板后转正

## 检查点（必须过用户确认）

- inventory 后：核对清单先给用户过目，**不擅自动结构**
- plan 落盘前：资源映射表先展示，用户说「换掉 X」就重排
- 涉及删除：永远列清单等确认

## 场景 few-shot

- 「给你个截图，这是老大的期末，给我出暑假快速学习方案」→ plan（诊断→映射→排期→落 OB）
- 「二宝要幼小衔接，英语还没启蒙」→ plan（幼儿启蒙区资源：牛津树/海尼曼/SSS/AlphaBlocks 排启蒙序列）
- 「云盘又乱了，整理下」→ inventory → 用户确认 → organize → catalog
- 「云盘里有什么数学资源」→ 直接查《阿里云盘索引》回答，不用扫盘
