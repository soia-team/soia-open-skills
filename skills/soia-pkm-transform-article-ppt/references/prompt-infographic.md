# Infographic Prompt

用于 PPT 媒体包中的独立信息图或重点简图。最终中文信息图优先由 HTML/CSS 或 PPT 排版截图生成；imagegen 只提供无字主视觉。

## 先确定信息任务

信息图只选一种主任务：

- `map`：展示章节、层级或知识地图。
- `flow`：展示从输入到输出的方向和检查点。
- `comparison`：区分易混概念或方案。
- `decision`：帮助读者做选择。
- `timeline`：展示真实有序阶段；没有可靠时间数据时不用。

不要把 60 个术语、完整文章和所有例子强塞进一张图。信息图应保留一条主线，完整内容回到 PPTX。

## Prompt 模板

```text
请把 source 转换为一张中文信息图。

画布：{canvas，默认 1080x1600 或用户指定}
主任务：{map | flow | comparison | decision | timeline}
主判断：{main_verdict}
受众：{audience}
核心信息块：{6-12 blocks}
主视觉素材：{asset_path_or_none}

结构要求：
1. 首屏 2 秒内能读到主判断。
2. 中部只有一条明确阅读路径；方向必须和语义一致。
3. 每个信息块包含短标题、解释和必要锚点，不堆长段正文。
4. 底部给读者下一步、判断边界或继续追问。
5. 来源信息简洁，不写运行元数据。

设计要求：
1. 中文由 HTML/CSS 或 PPT 排版，字号在目标尺寸下可读。
2. 图片不承载术语、数字、表格和来源。
3. 文字与视觉共用网格，不互相遮挡；不把图放进装饰卡片。
4. 使用 2-4 个语义色，不做单一色相堆叠和装饰渐变。
5. 标题、最长术语和英文缩写要实测断行。
```

## 验收

- 导出 PNG 并核对实际尺寸。
- 查看原始分辨率；检查文字、路径方向、间距和底部信息。
- 与 PPT 的 `main_verdict`、术语分组和风险口径一致。
- 若内容仍像海报而不是信息图，减少装饰、重做信息架构。
